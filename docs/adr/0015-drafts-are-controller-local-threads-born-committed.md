# Drafts are controller-local; threads are born committed

_Decided 2026-07-17. Supersedes [0010](./0010-bind-time-sessions-and-agent-lock.md)._

No worker state of any kind exists for a conversation until its first user message. A **draft** — chosen project, branch or worktree choice, agent and model/mode/effort/persona preferences, unsent message — lives only on the controller. There is no thread row, no git worktree, no ACP session, and no in-memory binding for a draft; `createThread` and `bindAgent` as pre-message operations are removed.

The first message runs one compound `startThread` operation on the worker: create the thread row → create the worktree / check out the branch (if chosen) → `session/new` at the effective cwd → apply draft preferences → persist `agentName` + `sessionId` → prompt. Threads are therefore born committed: every persisted thread has a locked agent and a session from birth. If the session or prompt fails mid-flight, the thread row and the user's message still persist, with the failure recorded as a conversation entry (per [0013](./0013-errors-and-titles-as-conversation-state.md)) so the user retries in place — a first message never silently vanishes.

Catalog for drafts comes from a **probe session**: a short-lived session created at the project directory solely to capture the catalog, then closed, never attached to a thread. The draft catalog is a best-effort preview, cached controller-side only and keyed by `(agentName, projectDir)` — explicitly not branch- or worktree-aware, and with no worker-side cache yet (both deliberate; the session created at first message is the source of truth and reconciles any staleness). Committed threads keep thread-scoped catalog reads from their own session, which the worker resumes lazily from the persisted `sessionId` when an operation needs it.

Trade-offs accepted: a fresh project's first thread pays two `session/new` calls (one probe, one real); drafts are invisible to other controllers (no row to sync); a probe at the project dir can show commands/skills that differ from the branch the first message eventually lands on. Rejected alternatives: a long-lived probe session per `(agentName, cwd)` (idle-session lifecycle — the orphan pain 0010 was written to kill, resurrected), and handing the probe session to the first thread that chats (reintroduces the stale-session-resume bug class this decision exists to delete).
